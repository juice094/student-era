1核心定义

宏病毒是利用 Office 宏语言（VBA）编写的恶意代码，寄生在文档或模板中，随文档打开执行。

宏病毒是利用 Office 宏语言（VBA）编写的恶意代码，寄生在文档或模板中，随文档打开执行，具有自我复制、感染其他文档及执行恶意操作的能力。

⚠️ 关键特征
一旦用户点击"启用内容"，AutoOpen / Document_Open 过程立即执行，病毒获得运行权限。Normal.dotm 全局模板被感染后，所有新建文档都会携带病毒代码。

2四大传播途径

宏病毒通过多种渠道扩散，了解传播方式是构建第一道防线的基础。

📨
邮件附件
最常见的传播方式，钓鱼邮件伪装成发票、简历等诱导打开。
💾
U盘/移动硬盘
插入已感染U盘后，宏病毒通过文档复制扩散，尤其在打印店、会议室等公共场景。
🌐
网络共享
企业局域网共享文件夹中的文档被感染后，所有访问者都面临风险。
⬇️
下载软件
从不明网站下载的 Office 模板或工具文档可能携带宏病毒。

阶段一：AutoOpen 触发

当用户打开文档并点击"启用内容"后，Office 立即执行 AutoOpen / Document_Open 过程。这是宏病毒的入口点。

Sub AutoOpen() Application.DisplayAlerts = False Call InfectSystem End Sub

阶段二：感染全局模板

Normal.dotm 是 Word 全局模板，所有新建文档都会继承它。病毒将自身代码复制到这里，实现"一次感染，终生带毒"。

Set nt = Application.NormalTemplate nt.VBProject.VBComponents("ThisDocument").CodeModule _ .AddFromString Me.CodeModule.Lines(1, 100)

阶段三：遍历感染文档

病毒扫描最近打开的文件列表、桌面、文档目录，将自身注入到其他 Office 文档中，实现横向扩散。

For Each doc In RecentFiles If doc.Path <> "" Then InfectDocument doc.Path End If Next doc

阶段四：执行恶意操作

最终阶段：下载后门、加密文件、窃取凭证，或启动隐藏的 PowerShell 连接远程服务器。

x = "powershell -enc ..." Shell x, vbHide

6宏病毒清除四步法

从紧急止损到根治清理的完整操作流程。

1

关闭宏

Word 选项 → 信任中心 → 宏设置 → "禁用所有宏，并发出通知"

2

重置模板

删除 %AppData%\Microsoft\Templates\Normal.dotm，重启 Word 自动生成干净模板

3

审查文档

用 olevba 批量提取宏源码，删除可疑模块或另存为 .docx（新版格式默认不保存宏）

4

升级补丁

保持 Office 和 Windows 更新，修补宏相关安全漏洞。企业环境建议组策略统一禁用宏，邮件网关预扫描附件。

💡 企业级建议
权限最小化 + 应用白名单 + EDR 行为监控 + 邮件网关预扫描。对于涉及财务、人事等敏感部门的文档，建议强制启用"仅受信任位置"的宏策略。

9课堂延伸问题

基于验证结论设计的开放性讨论题，点击展开参考答案。

▼
Q1：宏病毒会变异吗？变异后如何防范？
会的。现代宏病毒大量使用 VBA 混淆、P-Code Stomping、动态下载 Payload 等技术变异。防范不能仅靠特征码，需结合行为检测（如 MacroRaptor）和零信任宏策略。
▼
Q2：AI 生成的钓鱼邮件会让宏病毒更难防吗？
是的。AI 使钓鱼邮件的语言更自然、个性化更强，诱导启用宏的成功率可能上升。防御核心不再是"识别假邮件"，而是"默认不信任文档中的宏"。
▼
Q3：为什么企业组策略禁用宏后，仍有员工中招？
常见原因包括：① 员工使用个人设备绕过企业策略；② 文档被另存为受信任位置的文件；③ 社会工程学诱导员工手动更改宏设置；④ 利用 Office 已知漏洞绕过宏提示。防御需要技术+管理+培训三重措施。
▼
Q4：云办公（WPS/Office 365）能否杜绝宏病毒？
不能。云办公虽然增加了沙箱和实时扫描，但宏病毒的核心攻击面（VBA 执行环境）仍然存在。且云文档的协作共享特性反而可能加速病毒传播。关键仍是最小权限原则和用户安全意识。
课堂建议：Q1-Q2 适合作为"技术演进"辩论的开场；Q3-Q4 适合分组讨论（技术组 vs 管理组）。

10总结升华

安全意识是最好的杀毒软件。

不启用不明宏，不打开可疑件；

升级补丁要及时，安全意识挂心间。

🛡️
📎 核心要点回顾
宏病毒利用 Office VBA 自动执行机制，通过钓鱼邮件、U盘、共享文件夹传播。清除需禁用宏→重置模板→审查文档→升级补丁。防御核心是零信任宏策略：默认不信任任何文档中的宏代码。

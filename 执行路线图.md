## 二、串并行策略

### Week 1: Phase 1 基础能力（当前执行）

| 轮次 | 任务 | 依赖 | 并行度 |
|------|------|------|--------|
| **Round 1** | A1 登录页 + A2 UserStore + B1 角色定义 + C1 DictStore | 无 | 🔴 全并行（4 任务同时写代码） |
| **Round 2** | A3 路由守卫 + A4 Token 拦截器 + B3 v-permission 指令 | A2, B1 | 🔴 串行（必须等 Round 1 完成） |
| **Round 3** | B2 动态菜单 + D1 后台布局 + C2 字典 API | A3, B1 | 🟡 并行（3 任务同时） |
| **Round 4** | C3 字典页面 + D2 面包屑 | C1, C2, D1 | 🔴 串行（收尾） |

### Week 2-3: Phase 2 核心业务

| 轮次 | 任务 | 依赖 | 并行度 |
|------|------|------|--------|
| **Round 5** | 2-A 教师管理 + 2-B 课程管理（并行） | Phase 1 | 🟡 并行 |
| **Round 6** | 2-C 成绩管理（录入/查询/统计） | 2-A, 2-B | 🔴 串行 |

### Week 4: Phase 3 + Phase 4

| 轮次 | 任务 | 依赖 | 并行度 |
|------|------|------|--------|
| **Round 7** | 3-A 选课 + 4-A 操作日志（并行） | Phase 2 | 🟡 并行 |
| **Round 8** | 3-B 排课 + 4-B 导入导出（并行） | 3-A | 🟡 并行 |
| **Round 9** | 3-C 评教 + 4-C 大屏（并行） | 3-B | 🟡 并行 |
